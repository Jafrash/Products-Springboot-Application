# Product Management API (Spring Boot)

A simple RESTful API built with Spring Boot for managing product/employee records. This repository implements basic CRUD operations backed by an in-memory H2 database and exposes OpenAPI/Swagger documentation.

## Project overview

This is a minimal, example Spring Boot application that demonstrates how to:

- Model an entity with JPA and Lombok (`com.thehartford.Product.Model.Product`).
- Expose REST endpoints via a controller layer (`com.thehartford.Product.controller.ProductController`).
- Keep business logic in a service layer (`com.thehartford.Product.service.ProductService`).
- Use SpringDoc OpenAPI to provide API documentation.

The app is intentionally small and suitable for learning, demos, or as a starter template.



## Data model

Product entity (fields as implemented in `Product.java`):

- id (int) — autogenerated primary key
- name (String)
- department (String)
- salary (double)
- email (String)
- phoneNumber (String)
- address (String)

Example JSON representation:

```json
{
  "id": 1,
  "name": "Jane Doe",
  "department": "Engineering",
  "salary": 70000.0,
  "email": "jane.doe@example.com",
  "phoneNumber": "555-1234",
  "address": "123 Main St, Anytown, USA"
}
```


## REST API

Base path: `/products`

Endpoints:

- **POST** `/products`
  - Description: Create a new product record.
  - Request body: Product JSON (omit `id` — it is generated).
  - Response: Created Product (JSON)

- **GET** `/products`
  - Description: Retrieve all products.
  - Response: JSON array of products.

- **GET** `/products/{id}`
  - Description: Retrieve a product by its ID.
  - Response: 200 with product JSON if found, 404 if not.

- **PUT** `/products/{id}`
  - Description: Update an existing product (full replace semantics used here).
  - Request body: Product JSON (server sets `id` from URL).
  - Response: Updated product JSON.

- **DELETE** `/products/{id}`
  - Description: Delete a product by ID.
  - Response: 204 No Content on success.

Example curl commands (replace host/port as appropriate):

```powershell
# Create
curl -X POST "http://localhost:8082/products" -H "Content-Type: application/json" -d '{"name":"John Doe","department":"Sales","salary":50000,"email":"john.doe@example.com","phoneNumber":"555-0001","address":"1 Market St"}'

# Get all
curl http://localhost:8082/products

# Get by id
curl http://localhost:8082/products/1

# Update (PUT)
curl -X PUT "http://localhost:8082/products/1" -H "Content-Type: application/json" -d '{"name":"John Doe","department":"Sales","salary":52000,"email":"john.doe@example.com","phoneNumber":"555-0001","address":"1 Market St"}'

# Delete
curl -X DELETE http://localhost:8082/products/1
```
